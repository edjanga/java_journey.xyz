    The first chapter aims to present values, types and operators in JavaScript (JS).
    It starts off with an explanation of bits which is the way computers encode data. This bit (pun not intended) was cool as it teaches how to convert a number in bits.
    The idea is pretty simple: like the numerical system is based on powers of 10, the bits system is based on powers of 2, e.g. 1,2,4,8 and so on. These powers can be reported in a table as follows:
    Exponent01234567
    1248163264128
    11110000
    If the second row and third row are multiplied element wise and then summed up, one obtains 15. The bits representation of 15 is given by the last row.

    How cool is that??!

    After that quick intro, values are introduced. In JS, there are 4 types of values. Here is a quick overview of what they are and their characteristics

    Numbers

    • Special numbers: Infinity, -Infinity and NaN which stands for not a number, e.g. 0/0
    • These numbers do not behave like any others.

    Strings

    • Single quotes, double quotes, or backticks to mark strings, i.e. 'strings', "strings" `strings`.
    • Template literate: ${number} --> will convert number into a string at that location.

    Booleans values

    • true
    • false

    Undefined values

    • null and undefined: absence of meaningful (NaN Python).

    Operators

    Most of the operators work like in Python and are intuitive. However, there are some behaviours and features that are specific to JS. I have listed them below:
    • Unitary operator
      • typeof
      Returns the type of the subsequent value.
    • Binary operator
      • ==/!=
      Compares values. Note that values of the same type will have their content assessed against other while a conversion is first performed for values with different types, e.g. numbers == string: string is converted to number and if that converted string is equal number, then true is returned. Only NaN does not equal itself.
      ===/!==
      Compares vales without type coercion happening
    • Ternary (or conditional) operator
      • expression? true_return : false_return
      Compact way to return true_return if expression is true and false_return otherwise.

    Definition

    Type coercion
    Phenomenon that automatically converts a value type adequately in a binary operator if two different types are involved, e.g. 8*null --> 0 [null has been converted to 0]. Note that the conversion does not always work as expected, e.g. string + number ---> string [number is converted into a string and both strings are concatenated afterwards].
